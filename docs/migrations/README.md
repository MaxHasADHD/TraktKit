## Migration Process

1. Check the [CHANGELOG](../../CHANGELOG.md) for a summary of breaking changes
2. Read the relevant migration guide for detailed instructions
3. Update your code incrementally, testing as you go
4. Run your test suite to catch any issues
5. Build your project - the compiler will help identify remaining issues

## Version Support

TraktKit follows [Semantic Versioning](https://semver.org/):
- **Major versions** (e.g., 1.x → 2.x) may include breaking changes
- **Minor versions** (e.g., 3.1 → 3.2) typically include new features and may include breaking changes
- **Patch versions** (e.g., 3.1.0 → 3.1.1) include bug fixes only
