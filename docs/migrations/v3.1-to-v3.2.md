# TraktKit 3.2.0 Migration Guide

## Overview

TraktKit 3.2.0 introduces improved error type naming for better clarity and consistency, plus automatic token refresh support. This guide will help you update your error handling code.

## What's New?

### Automatic Token Refresh

TraktKit now automatically refreshes your OAuth tokens! This means:

- **No more 401 errors** - Tokens are refreshed automatically when they expire within 1 hour
- **Seamless retries** - If a request fails with 401, the token is refreshed and the request is retried
- **Zero configuration** - Works automatically once you're authenticated
- **Better UX** - Users never see authentication errors due to expired tokens

**No code changes required!** This feature works automatically. Your existing code will benefit from automatic token refresh without any modifications.

## What Changed?

### Error Type Renames

Two error types have been renamed to better reflect their purpose:

1. **`TraktSpecificError` → `TraktAPIError`**
   - These are HTTP errors returned by the Trakt API with custom status codes (420, 423, 426, 520-522)
   - The new name makes it clear these are API-level errors

2. **`TraktKitError` → `TraktClientError`**
   - These are SDK/client-side operational errors (missing configuration, user not authorized, etc.)
   - The new name distinguishes these from API response errors

## Migration Steps

### Step 1: Update Error Type References

#### TraktSpecificError → TraktAPIError

**Before:**
```swift
do {
    let movies = try await traktManager.movies.trending().perform()
} catch TraktSpecificError.accountLimitExceeded {
    print("You've reached the account limit")
} catch TraktSpecificError.accountLocked {
    print("Your account is locked")
} catch TraktSpecificError.vipOnly {
    print("This feature requires VIP")
} catch TraktSpecificError.cloudflareError(let statusCode) {
    print("Cloudflare error: \(statusCode)")
}
```

**After:**
```swift
do {
    let movies = try await traktManager.movies.trending().perform()
} catch TraktAPIError.accountLimitExceeded {
    print("You've reached the account limit")
} catch TraktAPIError.accountLocked {
    print("Your account is locked")
} catch TraktAPIError.vipOnly {
    print("This feature requires VIP")
} catch TraktAPIError.cloudflareError(let statusCode) {
    print("Cloudflare error: \(statusCode)")
}
```

#### TraktKitError → TraktClientError

**Before:**
```swift
do {
    try await traktManager.checkToRefresh()
} catch TraktKitError.missingClientInfo {
    print("Client ID/Secret not configured")
} catch TraktKitError.malformedURL {
    print("Invalid URL")
} catch TraktKitError.userNotAuthorized {
    print("User not logged in")
} catch TraktKitError.couldNotParseData {
    print("Failed to parse response")
} catch TraktKitError.invalidRefreshToken {
    print("Refresh token is invalid")
}
```

**After:**
```swift
do {
    try await traktManager.checkToRefresh()
} catch TraktClientError.missingClientInfo {
    print("Client ID/Secret not configured")
} catch TraktClientError.malformedURL {
    print("Invalid URL")
} catch TraktClientError.userNotAuthorized {
    print("User not logged in")
} catch TraktClientError.couldNotParseData {
    print("Failed to parse response")
} catch TraktClientError.invalidRefreshToken {
    print("Refresh token is invalid")
}
```

### Step 2: Update Type Checking

If you're checking error types with `is` or `as?`:

**Before:**
```swift
if error is TraktSpecificError {
    // Handle Trakt-specific errors
}

if let specificError = error as? TraktKitError {
    // Handle client errors
}
```

**After:**
```swift
if error is TraktAPIError {
    // Handle Trakt API errors
}

if let clientError = error as? TraktClientError {
    // Handle client errors
}
```

### Step 3: Update Generic Error Handlers

**Before:**
```swift
func handleError(_ error: Error) {
    switch error {
    case let apiError as TraktSpecificError:
        handleTraktSpecificError(apiError)
    case let clientError as TraktKitError:
        handleClientError(clientError)
    default:
        print("Unknown error: \(error)")
    }
}
```

**After:**
```swift
func handleError(_ error: Error) {
    switch error {
    case let apiError as TraktAPIError:
        handleTraktAPIError(apiError)
    case let clientError as TraktClientError:
        handleClientError(clientError)
    default:
        print("Unknown error: \(error)")
    }
}
```

## Complete Error Hierarchy

After migration, TraktKit has the following error types:

### Network/HTTP Layer
- **`TraktError`** (alias for `SwiftAPIClient.APIError`)
  - Standard HTTP errors: 400, 401, 404, 429, 500, etc.
  - Example: `.unauthorized`, `.notFound`, `.rateLimitExceeded`

- **`TraktAPIError`** (formerly `TraktSpecificError`)
  - Trakt's custom HTTP status codes
  - `.accountLimitExceeded` (420)
  - `.accountLocked` (423)
  - `.vipOnly` (426)
  - `.cloudflareError(statusCode:)` (520, 521, 522)

### Authentication Layer
- **`AuthenticationError`** (alias for `SwiftAPIClient.AuthenticationError`)
  - Token lifecycle management
  - `.tokenExpired(refreshToken:)`
  - `.noStoredCredentials`

- **`TraktTokenError`**
  - Device authentication flow errors
  - `.invalidDeviceCode`, `.expired`, `.denied`, etc.

### Client/SDK Layer
- **`TraktClientError`** (formerly `TraktKitError`)
  - SDK operational errors
  - `.missingClientInfo`
  - `.malformedURL`
  - `.userNotAuthorized`
  - `.couldNotParseData`
  - `.invalidRefreshToken`

## Example: Complete Before & After

### Before (v3.1.x)
```swift
@MainActor
class MovieViewModel: ObservableObject {
    @Published var movies: [TraktTrendingMovie] = []
    @Published var errorMessage: String?

    func loadMovies() async {
        do {
            let result = try await traktManager.movies.trending()
                .page(1)
                .limit(20)
                .perform()
            movies = result.object
        } catch TraktError.unauthorized {
            errorMessage = "Please log in"
        } catch TraktError.rateLimitExceeded {
            errorMessage = "Rate limit exceeded. Please wait."
        } catch TraktSpecificError.accountLimitExceeded {
            errorMessage = "Account limit reached"
        } catch TraktSpecificError.vipOnly {
            errorMessage = "VIP feature - please upgrade"
        } catch TraktKitError.userNotAuthorized {
            errorMessage = "Authentication required"
        } catch {
            errorMessage = "An error occurred: \(error.localizedDescription)"
        }
    }
}
```

### After (v3.2.0)
```swift
@MainActor
class MovieViewModel: ObservableObject {
    @Published var movies: [TraktTrendingMovie] = []
    @Published var errorMessage: String?

    func loadMovies() async {
        do {
            let result = try await traktManager.movies.trending()
                .page(1)
                .limit(20)
                .perform()
            movies = result.object
        } catch TraktError.unauthorized {
            errorMessage = "Please log in"
        } catch TraktError.rateLimitExceeded {
            errorMessage = "Rate limit exceeded. Please wait."
        } catch TraktAPIError.accountLimitExceeded {
            errorMessage = "Account limit reached"
        } catch TraktAPIError.vipOnly {
            errorMessage = "VIP feature - please upgrade"
        } catch TraktClientError.userNotAuthorized {
            errorMessage = "Authentication required"
        } catch {
            errorMessage = "An error occurred: \(error.localizedDescription)"
        }
    }
}
```

2. **Find:** `TraktKitError`
